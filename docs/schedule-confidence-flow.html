ShowTracker release intelligence

SQLite reconciles. Convex serves.

The new flow moves expensive provider and release matching out of normal app reads. Convex exports compact tracked-library rows, the SQLite reconciler turns provider events into release facts, and Convex imports only the deltas the UI needs.

The contract

Convex owns Auth, user tracking state, watched episodes, feed projections, and lightweight realtime reads.
SQLite owns Batch provider events, identity matching, release facts, deltas, audit issues, and run history.
UI consumes Precomputed Convex projections and schedule buckets, not raw provider scans.

End-to-End Data Flow

The important direction is one-way for release intelligence: provider data enters SQLite, SQLite creates compact facts, Convex applies those facts, and the app reads the projected result.

01

Convex exports tracked rows

scheduleConfidence:exportTrackedLibrary paginates feedProjections and returns user/show ids, watched count, remaining count, provider ids, and timestamps.

02

SQLite imports the library

import-convex upserts those rows into library_items. This is the reconciler's working set.

03

Provider events are hydrated

TVMaze, TMDB, and AniList events are normalized into provider_events, with provider-qualified ids and air dates.

04

SQLite reconciles release facts

The reconciler matches by ids first, records confidence, computes latest released and next scheduled episodes, then writes release_facts and convex_deltas.

05

Convex applies and UI reads

applyReleaseDeltas patches shows, user tracking signals, projections, and schedule buckets. Home and Schedule then read Convex.

What Each Store Is For

This is not a full migration away from Convex. It is a responsibility split that keeps high-cost reconciliation out of normal app traffic.

SQLite reconciler store

  • library_items: compact tracked-show snapshot imported from Convex.
  • provider_events: normalized release/schedule events from providers.
  • release_facts: one reconciled answer per canonical tracked item.
  • convex_deltas: payloads safe to send back to Convex.
  • audit_issues: missing provider links, title fallback, fetch failures, conflicts.
  • runs: scanned counts, changed facts, audit count, and run summary.

Convex app database

  • shows: catalog metadata plus reconciled release totals.
  • userShows: user-owned tracking state and newEpisodeSignalAt.
  • feedProjections: denormalized Home/Upcoming rows.
  • scheduleCache: compact future episode buckets by date and media type.
  • Auth, favorites, custom lists, watched episodes, and profile data remain here.

Edge Cases Covered

The reconciler is built around visible confidence and explicit failure modes. The app should not silently hide a tracked show because release data is incomplete.

Case Behavior now Where it lands
direct id Provider-qualified ids match directly, so the fact is trusted and imported. matchConfidence: "direct_id", then Convex patches show/projection state.
bridged id A linked provider id can bridge the match when the exact source id is missing. matchConfidence: "bridged_id", still eligible for Convex import.
title fallback Title-only matching is recorded but treated as low-confidence. An audit issue is emitted, and Convex skips the delta.
missing provider No provider match becomes a health issue instead of invisible absence. audit_issues.issue_type = "missing_provider_link".
global release Provider events are treated as global release evidence unless a provider supplies region context. Released episodes can set available_now even if the old schedule feed would miss them.
future episode Future scheduled episodes are distinct from episodes available now. nextScheduled updates scheduleCache, without forcing Home attention.
completed has new content If released episodes exceed watched count, completed shows re-enter attention. userShows.status moves back to watching or plan_to_watch.
schedule duplicates Future cache entries dedupe by route/provider show id plus season and episode. scheduleCache.episodes keeps one compact entry per episode.
bad importer Import/export calls require SCHEDULE_CONFIDENCE_IMPORT_TOKEN. Token mismatch throws before reading or mutating protected data.

UI Impact

The visible product effect is that Home, Watchlist, and Schedule can converge on the same release state without the user opening a detail page.

Home

Reads feedProjections by user, media, status, and sort indexes. newEpisodeSignalAt and homeSortAt pull newly available content back into attention.

Watchlist

Uses tracked show state and reconciled episode totals to calculate watched, total, remaining, and progress. Reconciled totals prevent stale completed state.

Schedule

Reads date-bucketed scheduleCache, then matches entries back to tracked projections using provider ids before falling back to title lookup.

Operational Runbook

These commands are the current repo surface. A future VPS job can wrap the same steps.

npm run schedule-confidence:init
npm run schedule-confidence:import
npm run schedule-confidence:reconcile -- --fetch-providers
npm run schedule-confidence:audit
npm run schedule-confidence:apply
npm run schedule-confidence:validate
  • init creates the local SQLite schema.
  • import pulls the compact tracked library from Convex.
  • reconcile --fetch-providers hydrates real provider events and computes facts.
  • audit writes inspectable health output.
  • apply imports compact deltas back into Convex.
  • validate runs deterministic fixture coverage for the core edge cases.
